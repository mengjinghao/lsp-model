# v1.0.9 综合更新总结

本 PR 包含 v1.0.7 ~ v1.0.9 的全部改动，并合并了 PR #1 的代码质量修复。

## v1.0.9：合并 PR #1 代码质量修复

### 3 个关键 Bug 修复（来自 PR #1）

1. **LogX 无限递归（CRITICAL）**
   - 问题：`LogX.w()` 的 catch 块调用 `LogX.w()` 自身，若 `XposedBridge.log` 抛异常则无限递归导致栈溢出
   - 修复：catch 块改为静默 `catch (_: Throwable) {}`
   - 影响：12 个模块的 LogX.kt

2. **WakeLockHook 线程爆炸**
   - 问题：每个 wakelock acquire 创建新 `Thread`，高频场景下 OOM
   - 修复：改用 `Executors.newSingleThreadScheduledExecutor` 单线程池
   - 影响：BatteryOptimizer_NoRoot + BatteryOptimizer_Root

3. **MicroXEnhancer ConfigManager 字段缺失**
   - 问题：v1.0.6 新增 7 个字段（autoRedPacket/autoTransfer/stepModifier 等）在 `getGlobalConfig()` 和 `applyMicroXConfig()` 中未同步，导致功能不生效
   - 修复：补全 KEY 常量 + 读写逻辑

### healthcheck.py 精度提升
- 健康分计算：工具类不计入分母（93.6% → 95.3%）
- UI bug 检测：`findall+find` 改为 `finditer+match.start`（精确定位）
- 增加 `encoding=utf-8` 解决跨平台解码

## v1.0.8：LSPatch 合规性改造

- xposedminversion 82 → 93（匹配 LSPatch v0.6.1+ API）
- XposedLoader 加 `packageName == "android"` 系统进程过滤
- XposedLoader 加 `isFirstApplication` 子进程过滤
- Manifest 补 FOREGROUND_SERVICE 权限
- 影响：8 个 NoRoot 版模块
