# 微X增强模块 — LSPatch修补教程

## 一、项目总览

### 适用平台
- **操作系统**: Android 8.1 ~ Android 15
- **目标应用**: 微信 8.0+ 、QQ 最新正式版
- **框架要求**: LSPatch（无需Root、无需解锁Bootloader）

### 免Root实现原理
本模块基于 **LSPatch** 框架运行。LSPatch通过在目标APK的字节码中注入加载器，使其在应用进程启动时加载Xposed模块。所有Hook逻辑完全运行在微信/QQ的**应用进程内存**中：

```
微信/QQ进程启动 → LSPatch注入的加载器启动 → 读取xposed_init → 加载MainHook
→ 各Hook子模块按需Hook目标方法 → 仅在进程内存中修改方法行为
```

**全程不涉及**：
- 系统分区读写（/system）
- 内核模块加载
- /proc、/sys节点操作
- 全局系统属性修改（build.prop）
- Root权限请求

### 两种运行模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **本地模式** | LSPatch管理后台常驻，动态注入已安装应用 | 日常使用，可随时开关模块 |
| **集成模式** | 将模块打包进微信/QQ APK，可独立分发 | 给他人使用，无需对方安装LSPatch |

## 二、本地模式操作步骤

### 前置条件
- 下载安装 [LSPatch](https://github.com/LSPosed/LSPatch/releases) v0.6+
- 已安装微信 (com.tencent.mm) 或 QQ (com.tencent.mobileqq)

### 步骤1：安装LSPatch
1. 从GitHub Release下载LSPatch APK
2. 安装到手机
3. 打开LSPatch，授予通知权限（用于保活）

### 步骤2：编译本模块
```bash
# 在项目根目录执行
./gradlew assembleRelease

# 或Windows下：
gradlew.bat assembleRelease
```
输出文件：`app/build/outputs/apk/release/app-release.apk`

### 步骤3：导入模块并修补
1. 打开LSPatch，进入「管理」页面
2. 点击右下角「+」按钮
3. 选择「从存储目录选择APK」→ 选择编译好的 `app-release.apk`
4. 点击「开始修补」
5. 修补完成后，选择修补模式：「本地模式」

### 步骤4：创建修补后的APK（给目标应用）
1. 在LSPatch「管理」页面点击右下角「+」
2. 选择「选择已安装应用」→ 选择「微信」
3. 修补模式选择「本地模式」
4. 等待修补完成
5. **卸载原版微信**（重要！签名不一致无法覆盖安装）
6. 安装修补后的APK（路径在 `/sdcard/Android/data/org.lsposed.lspatch/` 下）

### 步骤5：启用模块
1. 在LSPatch主页面，点击修补后的微信
2. 选择「模块作用域」
3. 勾选「微X增强」模块
4. 返回，启动微信
5. 进入微信后可看到「微X增强」设置入口

## 三、集成模式操作步骤

### 适用场景
- 给不需要LSPatch的用户直接使用
- 分发预配置版本

### 步骤1：用LSPatch打包
1. 打开LSPatch「管理」页面
2. 点击「+」→「选择已安装应用」→「微信」
3. 修补模式选择「**集成模式**」
4. 在弹出的模块选择中，勾选「微X增强」
5. 等待打包完成

### 步骤2：分发与安装
1. 打包后的APK在 `/sdcard/Android/data/org.lsposed.lspatch/` 目录
2. 将该APK发送给目标设备
3. 目标设备**不需要安装LSPatch**
4. 卸载原版微信后安装此APK即可

### 集成模式的局限
- 模块更新需要重新打包整个APK
- 无法动态开关模块（需要在打包时确定）
- 微信更新时需重新打包新版本

## 四、模块作用域设置说明

在LSPatch中，作用域决定模块在哪个应用中生效：

### 推荐作用域配置
本模块的 `AndroidManifest.xml` 已声明推荐作用域，LSPatch会自动识别：

```
✅ com.tencent.mm      (微信)
✅ com.tencent.mobileqq (QQ)
❌ 其他应用              (不需要勾选)
```

### 手动勾选
1. 进入修补后的微信 → 模块作用域
2. 确认「微X增强」已勾选
3. **不要**勾选其他不相关的模块到微信作用域（避免冲突）

## 五、功能使用指南

### 首次使用
1. 修补微信/QQ并安装后，打开应用
2. 在应用内找到「微X增强」设置入口（桌面也会生成快捷方式）
3. 进入设置面板：
   - **模块总开关**：控制所有功能的总开关
   - **广告净化**：各场景广告拦截开关
   - **消息增强**：防撤回、防删朋友圈、备份、自动回复
   - **界面美化**：气泡、字体、红点、Tab、背景
   - **隐私增强**：正在输入、已读状态、转发限制
   - **批量管理**：缓存清理、好友导出、僵尸粉、群管理
   - **安全适配**：绕过安全检测
4. 开启需要的功能后，部分功能需要重启微信/QQ生效

### 配置导入导出
- **导出**：点击「导出配置到文件」→ 文件保存至Download目录
- **导入**：将配置文件放入Download目录 → 点击「从文件导入配置」
- 配置文件名：`microx_enhancer_config.json`

### 自动回复关键词设置
自动回复的关键词需要手动配置，修改SharedPreferences：

```bash
# 通过adb设置自定义关键词（示例）
adb shell am broadcast -a com.microx.enhancer.SET_REPLY_RULE \
  --es keyword "在吗" \
  --es reply "在的，有什么事吗？" \
  --ez isGroup false
```

### 与ShizukuSceneFix等模块共存
本模块设计为不读取全局进程列表、不修改系统属性，与以下模块**理论上**不冲突：
- ShizukuSceneFix
- Game-Unlocker
- 其他LSPatch模块

如果遇到共存问题，请参考排错文档。

## 七、版本更新

### 模块更新步骤（本地模式）
1. 编译新版本APK
2. 在LSPatch中重新导入模块APK
3. 无需重新修补微信（模块以独立APK形式存在）
4. 重启微信即可

### 微信版本更新
1. 更新微信 → 微信更新后模块会失效
2. 在LSPatch中对新版微信重新修补
3. 重新启用模块作用域

---

## 八、删除模块

### 本地模式
1. 在LSPatch中移除微信的作用域
2. 删除修补后的微信APK
3. 安装原版微信 → 恢复

### 集成模式
直接安装原版微信APK覆盖即可。
